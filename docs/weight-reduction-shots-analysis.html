TL;DR — What I Found

🔴 "weight reduction shots" has had exactly 1 click in Google history (Sept 2025, 0 conversions)

This is an ultra-low-volume specific query. However, its sibling cluster — "weight loss shots", "weight loss injections", "shots for weight loss" — generates hundreds of clicks/month with strong conversion rates (8–21%). The analysis covers the entire cluster, not just this one term.

🔴 Search-WeightLoss campaigns burn $163 on injection queries with zero conversions

The "Search-WeightLoss-en-dt-us" and "Search-WeightLoss-en-mo-us" campaigns served "weight loss injections" 24 times, spent $163.46, and recorded 0 conversions. The same keyword in the Generic campaigns converts at 11–13%. This is a campaign/LP routing issue.

🔴 LP title exposes "Google Desktop" to users

The landing page HTML <title> is "Top 5 Weight Loss Medications Google Desktop - Top 5 Weight Choices". "Google Desktop" is visible in browser tabs and SERP snippets. One-line fix that affects all Google Desktop campaigns.

🟡 Injection traffic lands on dti=None pages with 3× lower CVR than r4 variant

Google injection keywords land on pages with no dti parameter: CVR 6–11%. The r4 variant achieves 16–28% CVR. Simply adding ?dti=r4 to Injections ad group final URLs could triple conversions from this cluster.

🟡 Ad says "shots" — LP never does. Ad copy is good; LP is the bottleneck

The Injections ad group has excellent shot-matching headlines: "Which Shot Works Best?", "Best Weight Loss Shots Of 2026". But the landing page uses zero shot/injection language in title, H1, or above-the-fold content. The ad wins the click; the LP loses the conversion.

🟢 "shots for weight loss" is cheapest and highest-converting query in the cluster: $7.04 CPA, 21.4% CVR

Among all injection/shot searches, "shots for weight loss" has the lowest CPA ($7.04) and 21.4% conversion rate. This exact phrase variant should be targeted explicitly with an exact match keyword and a dedicated LP.

Methodology Guide — How to Replicate

The 10-step keyword investigation process

Step A — SERP reconnaissance (WebSearch)

Search the plain keyword. Note: who appears, what page types dominate, does our domain appear and for which exact URL?

Also run: site:top5weightchoices.com "keyword" OR "synonym" to see which of our pages rank.

Key insight: The URL that appears in organic search is what users see — independent of what our ads point to. If our organic URL has "Google Desktop" in the title, users see it in the SERP snippet.

Step B — BQ: exact keyword query stats (ads_SearchQueryStats) 
-- Always use the service account: GOOGLE_APPLICATION_CREDENTIALS=.../weightagent-f7946872ebb1.json
SELECT
  qs.search_term_view_search_term AS search_term,
  qs.segments_search_term_match_type AS match_type,
  c.campaign_name, ag.ad_group_name,
  SUM(qs.metrics_clicks) AS clicks,
  SUM(qs.metrics_impressions) AS impressions,
  ROUND(SUM(qs.metrics_cost_micros)/1e6, 2) AS spend_usd,
  ROUND(SUM(qs.metrics_conversions), 2) AS conversions,
  ROUND(AVG(qs.metrics_ctr)*100, 3) AS ctr_pct
FROM weightagent.GoogleAds.ads_SearchQueryStats_4808949235 qs
JOIN weightagent.GoogleAds.ads_AdGroup_4808949235 ag
  ON qs.ad_group_id = ag.ad_group_id AND ag._DATA_DATE = ag._LATEST_DATE
JOIN weightagent.GoogleAds.ads_Campaign_4808949235 c
  ON qs.campaign_id = c.campaign_id AND c._DATA_DATE = c._LATEST_DATE
WHERE LOWER(qs.search_term_view_search_term) LIKE '%keyword%'
  AND qs.segments_date >= DATE_SUB(CURRENT_DATE(), INTERVAL 365 DAY)
GROUP BY 1, 2, 3, 4 ORDER BY clicks DESC LIMIT 30

If exact phrase has <5 clicks: Broaden to semantic cluster using multiple LIKE patterns. The cluster matters more than the exact phrase.

Watch for: Same keyword in different campaigns with wildly different CVR — that's a LP routing issue, not a keyword issue.

Step D — BQ: LP performance by dti variant (visits + conversions) 
SELECT
  v.dti, v.platform_id, v.device,
  COUNT(DISTINCT v.id) AS visits,
  COUNT(DISTINCT CASE WHEN c.funnel_step='other' AND c.funnel_step_description='Quiz Start' THEN c.id END) AS quiz_starts,
  COUNT(DISTINCT CASE WHEN c.funnel_step='step_3' THEN c.id END) AS goal_events,
  ROUND(COUNT(DISTINCT CASE WHEN c.funnel_step='step_3' THEN c.id END) * 100.0 /
    NULLIF(COUNT(DISTINCT CASE WHEN c.funnel_step='other' AND c.funnel_step_description='Quiz Start' THEN c.id END), 0), 2) AS cvr_pct,
  ROUND(SUM(SAFE_CAST(c.value AS FLOAT64)) / NULLIF(COUNT(DISTINCT v.id), 0), 4) AS epv
FROM weightagent.WeightAgent.visits v
LEFT JOIN weightagent.WeightAgent.conversions c ON v.id = c.visit_id
WHERE v.entered_at_date >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY)
  AND v.platform_id IN ('google', 'bing')
GROUP BY 1, 2, 3 HAVING COUNT(DISTINCT v.id) >= 10
ORDER BY visits DESC LIMIT 30

Key question: Is the dti that serves the keyword achieving the same CVR as the best-performing variant? dti=None is a red flag — it means no LP testing is happening for that traffic source.

Step E — Crawl OUR landing page (WebFetch)

Crawl the LP URL. Extract exactly:

  • The HTML <title> tag (not the H1 — the title shows in browser tab + SERP)
  • <meta name="description"> — present? What does it say?
  • H1 — first heading the user sees on page
  • Count exact keyword phrase occurrences in body
  • Count synonym/variant occurrences ("shots" if keyword is "weight reduction shots")
  • Partner list: exact order, scores, prices shown vs actual

The title and meta description are invisible in normal browsing but are what search engines and browser tabs show. They matter hugely for organic and for post-click credibility.

Step F — Crawl competitors (WebFetch)

Crawl top 2-3 SERP competitors. Priority: #1 organic result, best editorial comparison site (Top10, Yahoo, Forbes), any direct business-model competitor.

Note: Ro and Hims return 403. Forbes blocks Claude. Use PostgreSQL landscape data for them instead.

Extract from each: title, meta desc, H1, whether they use the keyword phrase, their partner order and lead pricing, trust signals, CTA copy.

Step G — PostgreSQL competitor landscape 
SELECT
  src.name, snap.page_title, snap.meta_description,
  jsonb_array_elements(snap.extracted_json->'partners')->>'rank' AS rank,
  jsonb_array_elements(snap.extracted_json->'partners')->>'canonical_name' AS partner,
  jsonb_array_elements(snap.extracted_json->'partners')->>'score' AS score
FROM competitor_landscape_snapshots snap
JOIN competitor_landscape_sources src ON src.slug = snap.source_slug
WHERE snap.snapshot_date = (SELECT MAX(snapshot_date) FROM competitor_landscape_snapshots)
ORDER BY src.name, (jsonb_array_elements(snap.extracted_json->'partners')->>'rank')::int;

Key question: Does our #1 match market consensus #1? If not — why? Affiliate deal? If yes — good signal.

Step H — Crawl #1 partner page (WebFetch)

Crawl the actual website of the partner we rank #1. Extract:

  • Their H1 and key marketing claims
  • First-month price AND recurring monthly price — these are often different
  • Trust signals: patient count, media mentions, reviews, certifications
  • Language they use for the service ("shots", "injections", "medications"?)

Cross-reference with our table: Are we showing their real pricing? Are we showing their strongest trust signals? Are we creating false expectations?
