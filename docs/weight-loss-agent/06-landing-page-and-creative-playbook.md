# Landing Page and Creative Playbook

## Goal

Create a clear framework for how the future agent should think about:

- landing pages
- comparison pages
- headlines
- descriptions
- creative assets
- message/offer fit

This playbook should help the agent diagnose not only what is underperforming, but why.

## Core Principle

The business is selling confidence and direction inside a confusing, high-stakes market.

Users are often:

- high intent
- comparison-oriented
- price-sensitive
- trust-sensitive
- worried about legitimacy
- overwhelmed by drug and brand terminology

Therefore, good performance requires more than “good ad copy.”

It requires alignment across:

- keyword intent
- ad promise
- landing-page promise
- ranking logic
- partner choice
- claim safety

## Part 1: Landing Page Audit Framework

The agent should evaluate landing pages across these dimensions:

### 1. Intent Match

Questions:

- Does the page answer the reason the user searched?
- Is it serving a branded query, ingredient query, price query, or comparison query correctly?
- Does it meet the user at the right stage of decision-making?

### 2. Price Clarity

Questions:

- Is price visible early enough?
- Is the page clearly built for price-sensitive users if that is the dominant segment?
- Are discounts, ranges, shipping, fees, and membership implications understandable?

### 3. Ranking Logic

Questions:

- Is the top-ranked provider obviously the one the business wants to promote?
- Does the order make sense to the user?
- Are there contradictions between text, price, and ranking?

### 4. Trust / Legitimacy

Questions:

- Does the page clearly communicate that the offers involve licensed clinicians and medical review where appropriate?
- Does it separate editorial comparison from medical advice?
- Are compliance-sensitive claims stated carefully?

### 5. Differentiation

Questions:

- Does the user understand how provider A differs from provider B?
- Are all brands described with nearly the same language?
- Is there a useful comparison angle beyond “good, affordable, fast”?

### 6. CTA Clarity

Questions:

- Is the primary CTA obvious?
- Is there confusion between compare, read review, and visit site flows?
- Is the page making the next step feel easy?

### 7. Cognitive Load

Questions:

- Is the page too dense?
- Does it mix too many ideas too early?
- Does it bury the commercial hierarchy?

### 8. Claim Safety

Questions:

- Are approval claims correct?
- Are compounded products described carefully?
- Does any line imply medical certainty or product equivalence without support?

### 9. Mobile Experience

Questions:

- Does the page communicate the key offer quickly on mobile?
- Are the ranking and CTA visible above the fold?
- Is the comparison still understandable on small screens?

## Part 2: What Makes a Bad Landing Page In This Vertical

A bad landing page may fail in any of these ways:

- it talks generally about weight loss when the user searched a specific brand
- it talks about “best plans” without clear comparison logic
- it hides price for a price-sensitive audience
- it ranks a partner in a way that feels arbitrary
- it uses medically risky or vague language
- it fails to explain differences between semaglutide / tirzepatide / brands
- it looks commercial but not trustworthy
- it looks trustworthy but not commercially decisive

## Part 3: Creative Asset Evaluation Framework

Every asset should be judged on both numbers and logic.

### Quantitative signals

- impressions
- CTR
- click-out rate
- purchase conversion rate
- effective CPA contribution
- partner-weighted value

### Qualitative signals

- logical clarity
- comparison logic
- intent match
- trust tone
- price/value relevance
- compliance safety
- distinctiveness

## Part 4: Types of Bad Assets

### Type A: Exposure With No Pull

Pattern:

- many impressions
- low CTR

Likely causes:

- weak hook
- generic wording
- wrong message for the query
- low salience vs competitors

### Type B: Clicks With No Purchase

Pattern:

- acceptable CTR
- low or zero purchase

Likely causes:

- curiosity wording
- wrong expectation set
- landing-page mismatch
- attracts researchers, not buyers

### Type C: Logically Weak Comparison Framing

Pattern:

- message sounds “marketing-ish” but not strategically smart

Example logic problem:

- “Compare raw weight loss plans” may sound relevant, but it narrows the user into the wrong frame if the user needs a broader comparison

### Type D: Dangerous Claim Asset

Pattern:

- claim sounds powerful
- may even improve CTR
- creates compliance or truth risk

Examples:

- “FDA-approved” used too loosely
- implying a compounded option is equivalent to a branded drug
- overclaiming results or guarantees

### Type E: Low-Signal Asset

Pattern:

- too little data

Rule:

- do not overfit weak-sample assets
- classify them as low confidence and keep them in a separate queue

## Part 5: Asset Replacement Logic

When the agent recommends replacing an asset, it should explain the replacement strategy.

Possible replacement strategies:

- make the comparison more explicit
- make the price angle more explicit
- increase trust / legitimacy framing
- reduce vague healthcare jargon
- make the user outcome clearer
- align headline and landing-page story
- remove risky claim language

## Part 6: Questions The Agent Should Ask Before Rewriting Copy

1. What query or audience is this asset actually serving?
2. Is the user trying to compare providers, find the cheapest option, or validate trust?
3. Is the current line attracting the wrong user?
4. Does the landing page fulfill the promise this line makes?
5. Is the line logically sharp, or just semantically close to the topic?
6. Is there any medical/compliance phrasing risk?

## Part 7: Recommended Output Format For Asset Analysis

For each asset, the agent should return:

- asset text
- where it runs
- impressions
- CTR
- purchase contribution
- confidence level
- diagnosis
- why the diagnosis matters
- keep / pause / test recommendation
- rewrite suggestion

## Part 8: Comparison-Site-Specific Recommendations

Because the site is a comparison property, the agent should also evaluate:

- whether the number-one pick is clearly justified
- whether cheaper brands are surfaced clearly enough
- whether the page explains “why this one vs that one”
- whether rankings feel commercial but still believable
- whether users can quickly identify the best fit for their situation

## Part 9: Landing Page Themes To Investigate

The meeting explicitly hinted that the site may need stronger guidance around:

- price
- best value
- support level
- strength of offer
- holistic programs vs medication-first programs
- trust / legitimacy

The agent should therefore test themes such as:

- cheapest serious option
- best overall value
- strongest medical-first option
- best all-inclusive support option
- best fast-start option

## Part 10: Deep Page Research Process

For any important landing page, the agent should:

1. capture the page
2. extract visible copy
3. identify CTA hierarchy
4. identify ranking / recommendation logic
5. identify price signals
6. identify trust signals
7. identify risky wording
8. map keyword intent to page sections
9. compare against competitor pages
10. produce a rewrite and experiment plan

## Part 11: What “Good” Should Look Like

A strong landing page / creative system in this vertical should:

- make the comparison easy
- reduce confusion
- surface price and fit clearly
- feel medically responsible
- help users choose
- align tightly with search intent
- avoid careless medical claims

That is the standard the future agent should optimize for.
